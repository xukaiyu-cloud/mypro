# 桌面端音乐播放器 —— 项目开发报告

---

## 一、项目当前进度描述

本项目是一个基于 Electron + Vue 3 的桌面端音乐播放器，已从零搭建为具备完整前后端架构的可运行应用。当前处于 **Alpha 内测阶段**，核心播放功能和搜索系统已打通，正在持续打磨细节体验。

**已完成骨干功能**：用户注册/登录认证体系；本地 SQLite 数据库存储歌曲与歌单；酷狗/网易云双平台歌单链接解析导入；跨平台统一搜索（本地曲库 + 网易云 + 酷狗），支持拼音首字母、错别字容错、自然语言语义搜索（如"安静的治愈音乐"）；搜索结果分类展示（综合/单曲/歌单/歌手/专辑/歌词/视频），歌单卡片点击查看详情、右键菜单添加到我的歌单；播放控制（播放/暂停/上下曲/进度条拖拽/音量调节/播放模式切换）；Apple Music 风格沉浸式全屏播放器，专辑封面动态取色生成渐变背景；实时歌词滚动显示；本地文件夹扫描导入 MP3 文件；好友系统与一起听房间（WebSocket 实时同步）；播放历史与收藏管理。

**近期集中修复的 Bug 链**：歌单详情弹窗歌曲列表不渲染（服务端 id 硬编码 0 导致 Vue key 重复）、SongTable 的 CSS contain:strict 导致高度塌陷、搜索结果所有歌曲 id 为 0 导致点击任意歌曲都播放第一首、跨歌单同名 ID 导致播放错位、鼠标滚轮失效（SongTable 缺少 overflow-y:auto）、远程封面图 Canvas 跨域拦截导致沉浸式背景色不变化等，均已逐一修复。

**待完善方向**：多平台音源播放稳定性（部分歌曲因版权无法获取播放链接）、搜索建议联想词下拉、搜索结果内直接操作（加入歌单/收藏）、UI 动画细化和深色主题适配。

## 二、技术选型说明

### 前端技术栈

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 桌面容器 | Electron | 31.x | 跨平台桌面窗口、系统托盘、IPC 通信 |
| 前端框架 | Vue 3 | 3.5.x | Composition API、响应式数据驱动 |
| 类型系统 | TypeScript | 5.6.x | 全量类型约束、编译期错误检查 |
| 构建工具 | Vite + electron-vite | 5.4 / 3.1 | HMR 热更新、主进程/预加载/渲染三端构建 |
| 状态管理 | Pinia | 2.2.x | 播放器、歌单、收藏、好友等模块化 Store |
| 路由 | Vue Router | 4.4.x | Hash 模式、路由守卫鉴权 |
| UI 组件库 | Element Plus | 2.9.x | 对话框、标签页、按钮、上传等组件 |
| HTTP 客户端 | Axios | 1.7.x | 后端 API 调用 |
| 实时通信 | Socket.IO Client | 4.8.x | 一起听房间状态同步 |
| CSS 预处理 | Sass | 1.80.x | 嵌套、变量、混入 |

### 后端技术栈

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| HTTP 服务 | Express | 4.21.x | RESTful API 路由 |
| 数据库 | sql.js (SQLite) | 1.10 | 本地嵌入式数据库，无需安装 |
| 认证 | JWT + bcryptjs | 9.0 / 2.4 | Token 鉴权、密码哈希 |
| 网页解析 | Cheerio | 1.2 | HTML 解析（歌单链接抓取） |
| 实时通信 | Socket.IO | 4.8.x | WebSocket 房间管理 |
| 文件上传 | Multer | 1.4 | multipart/form-data 解析 |
| 网易云加密 | crypto (内置) | - | AES-128-CBC + RSA 实现 weapi 参数加密 |

### AI 工具

- **Codex CLI**（OpenAI）：作为本项目的核心 AI 编程助手，全程辅助需求分析、代码生成、Bug 诊断和架构决策
- 服务端集成 OpenAI API（`server/src/services/ai.ts`）：实现搜索关键词的**自然语言意图解析**（将"想听安静的纯音乐"解析为情绪标签 + 风格关键词），驱动语义搜索

## 六、风险识别与调整计划

### 已遇到的困难

| 困难 | 影响 | 解决方案 |
|------|------|----------|
| 网易云 weapi 加密 | 无法直接调用搜索 API | 逆向 AES + RSA 实现完整加密层（`ncm.ts`），现已可正常搜索和获取播放链接 |
| 远程封面 Canvas 跨域 | 沉浸式背景色不变 | 添加 `crossOrigin='anonymous'` + 服务端图片代理 `/api/proxy` 双重保护 |
| 多源歌曲 ID 冲突 | 点击歌曲 A 播放歌曲 B | `dedupeSongs()` 去重后统一分配 1,2,3… 递增 ID；`player.play()` 改为 id+title 复合匹配 |
| CSS contain 属性 | SongTable 完全不可见 | 从 `contain:strict` 改为 `contain:layout style paint`（移除 size），最终回退到简洁版 |
| 文件编码损坏 | 中文字符显示为乱码 | 全部文件使用 `System.Text.Encoding.UTF8` 字节级读写 |
| Windows GPU 崩溃 | Electron 启动即退出 | 添加 `--disable-gpu --disable-gpu-sandbox` 命令行参数 |
| PowerShell 正则转义 | 字符串替换失败 | 改用 `[regex]::Escape()` + indexOf 定位 + 子串拼接 |

### 当前风险点

| 风险 | 等级 | 调整计划 |
|------|------|----------|
| 网易云 API 反爬升级 | 中 | 定期更新加密参数，准备多个 User-Agent 轮换 |
| 部分歌曲无版权链接 | 高 | 实现多平台自动换源（酷狗→网易云→本地），增加"仅试听 30s"提示 |
| SQLite 并发写入 | 低 | 已有 persist() 机制，后续考虑引入 WAL 模式 |
| Electron 打包体积过大 | 低 | 预留 electron-builder 配置，排除 node_modules 中无关文件 |
| 跨平台兼容性 | 中 | 当前仅 Windows 测试，macOS/Linux 需验证文件路径和系统 API |
