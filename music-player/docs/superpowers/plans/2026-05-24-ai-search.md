# AI 语义搜索 — 实现计划

> **Goal:** 在搜索框中实现自然语言 AI 搜索，用户可用日常语言描述想听的歌，系统通过 LLM 解析意图后搜索本地+在线曲库返回结果。

> **Architecture:** SearchBar 检测自然语言 → `/api/search/ai` → LLM 意图解析 → 分派搜索策略（本地 MySQL + 网易云 API）→ 结果融合返回 → SearchPage 展示 AI 解释 + 结果列表。

> **Tech Stack:** Node.js/Express (后端), Vue 3 + TypeScript (前端), OpenAI-compatible API (LLM), 复用现有 ncm.ts 搜索 + MySQL 查询

### Task 2: 搜索路由

**Files:**
- Create: `D:\aiPro\pro2\music-player\server\src\routes\search.ts`

- [ ] **Step 2.1: 实现 POST /api/search/ai**

接收 `{ query, userId }`：
1. 调用 `ai.parseInt(query)` → intent
2. 根据 `intent.type` 执行搜索：
   - `keyword` → 本地 SQL LIKE + 网易云 searchSong
   - `artist_rec` → 网易云 searchSong(referenceArtists + genres)
   - `genre_mood` → 本地 + 网易云 searchSong(genres + mood)
   - `history_lookup` → MySQL play_history 查询
   - `playlist_gen` → LLM 生成歌单建议 → 网易云逐首搜索
3. 结果去重（title+artist 标准化比较）
4. 返回 `{ query, explanation, results, searchType: "ai" }`

- [ ] **Step 2.2: 实现 POST /api/search/quick**

接收 `{ query }`：
1. 并行：本地 MySQL LIKE + 网易云 searchSong
2. 合并去重
3. 返回 `{ query, results, searchType: "normal" }`

### Task 4: 前端 API 封装

**Files:**
- Create: `D:\aiPro\pro2\music-player\src\renderer\api\search.ts`

- [ ] **Step 4.1: 创建 search.ts API 模块**

```typescript
import http from './index'
import type { ApiResponse } from '../types/api'
import type { Song } from '../types/song'

export interface AiSearchResult {
  query: string
  explanation: string
  results: Song[]
  searchType: 'ai' | 'normal'
}

export const searchApi = {
  aiSearch: (query: string) =>
    http.post<ApiResponse<AiSearchResult>>('/search/ai', { query }).then(r => r.data),

  quickSearch: (query: string) =>
    http.post<ApiResponse<{ query: string; results: Song[]; searchType: string }>>('/search/quick', { query }).then(r => r.data),
}
```
